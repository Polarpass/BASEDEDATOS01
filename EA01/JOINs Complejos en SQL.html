🔄 SELF-JOIN

Una tabla se une consigo misma

🎯 Concepto: JOIN sobre la misma tabla utilizando alias diferentes. Útil para relaciones jerárquicas como empleados y managers.

-- Listar empleados con el nombre de su manager
SELECT e.first_name || ' ' || e.last_name AS "EMPLEADO",
       m.first_name || ' ' || m.last_name AS "MANAGER"
FROM employees e
LEFT JOIN employees m ON e.manager_id = m.employee_id
WHERE m.employee_id IS NOT NULL
FETCH FIRST 5 ROWS ONLY;

⚖️ NON-EQUI JOIN

Condiciones distintas a igualdad (=)

🎯 Concepto: JOIN basado en rangos, mayor que, menor que, BETWEEN. No usa el operador de igualdad.

-- Empleados cuyo salario está dentro del rango de su puesto
SELECT e.last_name AS "EMPLEADO",
       e.salary AS "SALARIO",
       j.job_title AS "PUESTO",
       j.min_salary AS "MÍN_PUESTO",
       j.max_salary AS "MÁX_PUESTO"
FROM employees e
JOIN jobs j ON e.salary BETWEEN j.min_salary AND j.max_salary
WHERE e.job_id = j.job_id
FETCH FIRST 4 ROWS ONLY;

💡 Ejemplo alternativo: ON e.salary > j.min_salary AND e.salary < j.max_salary

-- Todos los departamentos, tengan o no empleados
SELECT d.department_name AS "DEPARTAMENTO",
       COUNT(e.employee_id) AS "TOTAL_EMPLEADOS"
FROM departments d
LEFT JOIN employees e ON d.department_id = e.department_id
GROUP BY d.department_name
ORDER BY COUNT(e.employee_id) DESC
FETCH FIRST 5 ROWS ONLY;

-- Ubicaciones y sus departamentos (incluye ubicaciones sin departamento)
SELECT l.city AS "CIUDAD",
       d.department_name AS "DEPARTAMENTO"
FROM departments d
RIGHT JOIN locations l ON d.location_id = l.location_id
ORDER BY l.city
FETCH FIRST 6 ROWS ONLY;

Todos los registros de ambas tablas

-- Unión completa: empleados sin departamento + departamentos sin empleados
SELECT e.last_name AS "EMPLEADO",
       d.department_name AS "DEPARTAMENTO"
FROM employees e
FULL OUTER JOIN departments d ON e.department_id = d.department_id
WHERE e.department_id IS NULL OR d.department_id IS NULL
ORDER BY d.department_name NULLS FIRST
FETCH FIRST 5 ROWS ONLY;

✖️ CROSS JOIN

Producto cartesiano

🎯 Concepto: Combina cada fila de la tabla A con cada fila de la tabla B. Útil para generar combinaciones o datos de prueba.

-- Combinar regiones con países (ejemplo simplificado)
SELECT r.region_name, c.country_name
FROM regions r
CROSS JOIN countries c
WHERE r.region_id = c.region_id
FETCH FIRST 5 ROWS ONLY;

-- Otra forma: SIN WHERE genera todas las combinaciones posibles
SELECT e.last_name, d.department_name
FROM employees e
CROSS JOIN departments d
FETCH FIRST 5 ROWS ONLY;

🔗🔗🔗 JOIN con MÚLTIPLES TABLAS (3 o más)

Combinando información de employees, departments, locations, countries
-- Reporte completo: empleado → departamento → ciudad → país
SELECT e.first_name || ' ' || e.last_name AS "EMPLEADO",
       d.department_name AS "DEPARTAMENTO",
       l.city AS "CIUDAD",
       c.country_name AS "PAÍS"
FROM employees e
JOIN departments d ON e.department_id = d.department_id
JOIN locations l ON d.location_id = l.location_id
JOIN countries c ON l.country_id = c.country_id
ORDER BY e.last_name
FETCH FIRST 6 ROWS ONLY;
